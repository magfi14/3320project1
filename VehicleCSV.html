create(self, prx='')
- Creates a new vehicle via tiered recursion to control the flow of user-input while mitigating technical error & achieving the user's end goal by the following:
    - The outer condition checks for if the user input is a single-word command as follows:
        - C: Create
        - R: Replace
        - U: Update
        - D: Delete (Removable Condition)
        - N: Size
        - W: Wipe Clean (Removable Condition)
    - If the outer condition reins true, the inner condition checks for whether the command given is in the removables list. If not, the following happens:
        - The first tiered case checks for if the command is for 'RETRIEVE' / 'R'. If so, the entirety of the existing contents of the CSV file are displayed to the user in a neatly-formatted DataFrame.
        - The second tiered case checks for if the command is for 'UPDATE' / 'U'. If so, the createx() function is called to attain the row number to update as well as the data to be input. Upon successful completion of that loop, the assigned row is updated with the new vehicle data on the condition that there are vehicles in the CSV file already. If not, the create() function is called again but under the 'CREATE' / 'C' parameter to create a new vehicle.
        - The third tiered case checks for if the command is for 'NUMBER OF ROWS' / 'N'. If so, the number of vehicles in the CSV file is returned to the user on the terminal.
        - If the prompt is not in the data manipulation command list ['R', 'U', 'N'], the createx() function is called to fill in the details for a new vehicle. If the prompt is 'CREATE' / 'C', the list of attributes is retrieved (populated by createx()) which is added to the end of the CSV file via the data manager.
    - If the command is in the removable command list ['D', 'W'], the createx() function is not called at all. What happens from here is beholden on whether or not at least one vehicle exists in the CSV file.
        - If there is at least one vehicle in the CSV file & the command is 'DELETE' / 'D', the user will then be given an opportunity to pick which row gets deleted. The data is passed into the data manager, where the delete() function is called inside the data manager to delete the row from the CSV file.
        - If there is at least one vehicle in the CSV file & the command is 'WIPE' / 'W', the entire CSV file will be cleared & all existing vehicles will be removed.
        - If there are no vehicles in the CSV file, the create() function is called from the very beginning.
    - Finally, if the command is 'Q', the program stops comopletely. All vehicle creation is halted at that specific point.
 
### Parameters:
- prx (str): The command to be used to control the recursion tiers & mitigate technical error
createx(self, key='CLASS', step=1, prompt='C', rowno=-1)
- Uses layered, parametric base cases to control & regulate data input via user-input to the terminal.
- If the prompt is 'U', the terminal will ask the user which row to change. If it is within the bounds, that row in the CSV file will be replaced with the data follow.
- If the prompt is 'C', the terminal will ask the user in layered order of base case according to the attribute's name as follows:
    - -> Class of vehicle. A conditional block is used to test whether the size class of the vehicle is in the predefined list of size classes. If it is not, the createx() function is called again with the same step number. If the user input of size class is in the list of size classes, the size class is saved with the user input & the step is incremented in the recursive createx() call for the second ordered layer.
    - -> Type of vehicle. A conditional block is used to test whether the type of vehicle is in the predefined list of types. If it is not, the createx() function is called again with the same step number. If the user input of the type is in the list of types, the type is saved with the user input & the step is incremented in the recursive createx() call for the third ordered layer.
    - For each subsequent recursive layer, each user input saves that attribute to that specific attribute's variable, incrementing the step & moving to the next subsequent recursive layer. At the last layer, the user assigns the vehicle to a fleet number before it returns the number of the row to change in the case that the initial prompt was set to 'U' instead of 'C'.
 
### Parameters:
- key (str, optional): The attribute's key in the list of keys
- step (int, optional): The iterator which increments & decrements to move up & down the layers recursively, which is beholden on the nature of the user input
- prompt (str, optional): The type of prompt, whether to create a new entry in the CSV file or to update an existing entry in the CSV file
- rowno (int, optional): The row to update in the event that the prompt is 'U' for 'update' instead of 'C' for 'create'
 
### Returns:
- int: The row number to be saved for each recursive layer of user-input, assuming 'U' condition
